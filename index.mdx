brenlabs is an evaluation platform for AI coding agents. You bring a product
to test against, an agent configuration (Claude Code, Codex, …), and a
scenario describing the work the agent should do. brenlabs spins up an
isolated sandbox, runs the agent against the scenario, streams the full
turn-by-turn transcript, and judges the result against the success criteria
you authored.

It exists because coding agents behave differently depending on which skills,
MCP servers, and system prompts they're configured with — and the only way
to find out which combination actually works is to run them.

## Who this is for

- Teams building **products that AI agents will use** — APIs, SDKs, internal
  tools — and want to know whether agents can actually accomplish real tasks
  against them.
- Teams **authoring agent configurations** — skills, MCP servers, system
  prompts — who need a way to A/B test changes before shipping them.
- Engineers debugging **why an agent did the wrong thing** in production —
  brenlabs preserves the full transcript, every tool call, every thinking
  block, and the judge's per-criterion analysis.

## How it works

<Steps>
  <Step title="Describe what you're testing">
    Register the product (URL + auth), define one or more environments
    (staging, prod, etc.), and configure an agent with the skills and MCP
    servers it should have access to.
  </Step>
  <Step title="Write a scenario">
    A short markdown brief telling the agent what to do, plus a success
    criteria block that's hidden from the agent and shown only to the judge.
  </Step>
  <Step title="Run it">
    brenlabs creates a Modal sandbox from the right base image, materialises
    the scenario, skills, and MCP config, then executes the agent. The full
    transcript streams to your browser in real time.
  </Step>
  <Step title="Read the verdict">
    A separate judge model reviews the transcript against the criteria,
    producing a structured verdict — pass/fail, confidence, per-criterion
    breakdown, severity-sorted flags.
  </Step>
</Steps>

## API access

brenlabs exposes everything through a REST API at `https://api.bren.run`.
Anything you can do in the dashboard, you can script. See the
[API reference](/api-reference/openapi) for the full catalogue.
